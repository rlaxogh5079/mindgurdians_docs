# 개발 과정

---

## 멀티 플랫폼 클라이언트 개발

Flutter를 사용하여 환자·의료진용 애플리케이션을 단일 코드베이스로 개발을 진행합니다.

Android, iOS, Windows, MacOS, Linux, Web 등 다양한 플랫폼에서 동일한 사용자 경험(UX)를 제공합니다.

환자, 의료진 사용자 인터페이스(UI)와 환자 관리 기능(진료 기록 확인, 알림 수신 등)을 제공합니다.

실시간 생체신호 시각화를 위한 커스텀 차트·그래프 구현합니다.

WebRTC 화상통화 UI, FCM 알림 UI 통합합니다.

보안 강화를 위해 HTTPS 기반 API 통신을 진행하며, 민감한 데이터는 암호화 하여 저장합니다.

---

## 서버 개발

스마트밴드·앱·의료진 시스템을 연결하는 핵심 백엔드 플랫폼을 개발합니다.

Spring Boot 기반 RESTful API 서버를 구축합니다.

JWT 기반 인증/인가 체계를 적용합니다.

사용자·역할 기반 접근 제어(Role-Based Access Control) 기능을 제공합니다.

TLS(HTTPS) 인증서를 적용하고, 민감한 데이터는 AES-256 알고리즘을 통해 서버측에서 암호화를 진행합니다.

Flutter 클라이언트·스마트밴드·대시보드 간 데이터 교환 처리를 진행합니다.

!!! note

    AES-256 알고리즘은 대표 알고리즘을 적용하였습니다. 추가 조사 후에 다른 알고리즘으로 변경할 수 있습니다.

---

## 데이터 저장 및 분석 플랫폼

스마트밴드 생체신호부터 정형 데이터까지 통합 관리되는 데이터 생태계입니다.

데이터 저장은 다음과 같이 진행합니다.

RDBMS - PostgreSQL

- 환자 프로필, 진료 기록, 상담 내역, 처방 정보 등 정형 데이터를 관리합니다.
- 트랜잭션 무결성을 확보합니다.
- 알람 로그를 저장 및 추적합니다.

NoSQL - MongoDB

- 화상 진료 세션 로그, 이벤트 로그 등 비정형 JSON 데이터를 저장합니다.
- 빠른 쓰기 성능과 수평 확장성을 제공합니다.

TSDB - InfluxDB

- 스마트밴드 생체신호를 저장합니다.
- Flux 쿼리를 활용한 시간 기반 집계를 진행합니다.
- 실시간·장기 추세 분석 기반의 의료진 모니터링 서비스를 구현합니다.
- Grafana와 연동해 의료진 대시보드를 시각화합니다.

## 푸시 알림 시스템 (Firebase Cloud Messaging, FCM)

사용자 맞춤형 실시간 알림을 제공합니다.

- 진료 예약, 상담 요청, 복약 알림 등 환자·의료진 맞춤 알림을 제공합니다.
- Flutter 기반 모바일·데스크탑 푸시 알림을 구현합니다.
- 의료진 → 환자, 환자 → 의료진으로 흐름을 분리합니다.
- 알림 내역은 PostgreSQL에 저장하여 이력 관리 및 분석이 가능합니다.

## WebRTC 표준기술 기분 원격 화상진료

실시간 의료 상담을 위한 고품질 실시간 영상·음성 통신 기능입니다.

- WebRTC 표준 기반 P2P 또는 SFU(MCU) 구조 중 선택하여 진행합니다.
- E2EE(종단 간 암호화)를 적용합니다.
- Flutter 클라이언트·음성 통신 기능.웹 브라우저 간 상호 호환성을 확보합니다.
- 대역폭 조절·화질 제어 등 실시간 최적화 옵션을 포함합니다.
